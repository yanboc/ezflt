# 贡献指南

欢迎为 ezflt 项目做出贡献！

## 开发环境设置

1. Fork 项目
2. 克隆你的 fork
3. 按照 [快速开始](getting_started.md) 设置开发环境
4. 运行测试确保一切正常

## 代码规范

- 遵循 PEP 8 代码风格
- 使用类型提示
- 编写清晰的文档字符串
- 保持代码简洁和可读性

## 测试

在提交代码前，请确保所有测试通过：

```bash
pytest tests/ -v
```

## 提交规范

- 使用清晰的提交信息
- 每个提交应该是一个逻辑单元
- 提交前运行测试和代码检查

## 文档

- 更新相关文档
- 为新功能添加使用示例
- 保持 API 文档的准确性
